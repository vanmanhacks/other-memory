36|## What It Does
37|
38|Other Memory exposes two MCP tools any AI agent can call:
39|
40|| Tool | Backend | What It Does |
41||------|---------|--------------|
42|| `web_search` | [SearXNG](https://github.com/searxng/searxng) | Meta search across Google, Bing, DuckDuckGo, Brave, Mojeek, Qwant, Startpage, GitHub, Wikipedia |
43|| `web_crawl` | [Crawl4AI](https://github.com/unclecode/crawl4ai) | JavaScript-rendered page extraction via headless browser. Returns clean markdown, plain text, or raw HTML |
44|
45|Both tools are cached through Redis (1-hour TTL), deduplicated, rate-limited, and SSRF-protected. All zero-cost, zero-key, running entirely on your own metal.

49|## Architecture
50|
51|```
52|┌──────────────────┐     MCP stdio      ┌──────────────────────┐
53|│  Hermes Agent    │ ──────────────────→│   MetaMCP Bridge     │
54|│  (or any MCP     │                    │   (Python, async)    │
55|│   client)        │                    │                      │
56|└──────────────────┘                    │  web_search()  ──────┤
57|                                        │  web_crawl()   ──────┤
58|                                        └──────┬─────────┬─────┘
59|                                               │         │
60|                                    ┌──────────▼──┐  ┌───▼──────────┐
61|                                    │   SearXNG    │  │  Crawl4AI    │
62|                                    │   :8888      │  │  :11235      │
63|                                    │              │  │              │
64|                                    │ Google, Bing,│  │ Headless     │
65|                                    │ DDG, Brave,  │  │ browser      │
66|                                    │ GitHub, WP…  │  │ (Playwright) │
67|                                    └──────┬───────┘  └──────────────┘
68|                                           │
69|                                    ┌──────▼───────┐
70|                                    │    Redis      │
71|                                    │    :6380      │
72|                                    │  (cache, 1h)  │
73|                                    └──────────────┘
74|```
75|
76|**The bridge (MetaMCP)** is a single Python file, `metamcp/server.py`, that translates the MCP protocol to SearXNG and Crawl4AI HTTP APIs. It handles:
77|
78|- **Caching**: SHA-256 keyed Redis cache with 1-hour TTL. Repeated queries hit cache, not upstream engines.
79|- **Deduplication**: Multiple engines may return the same URL; results are deduplicated by URL before being returned.
80|- **Graceful degradation**: If Redis is down, the bridge keeps working (just without caching). If SearXNG is down, it returns a clear error, not a crash.
81|- **SSRF protection**: Blocks internal IPs (10.x, 172.16-31.x, 192.168.x), metadata endpoints (169.254.169.254), and non-HTTP schemes.
82|- **Token budget awareness**: Crawl output is truncated at 50K characters with a midpoint cut (keeps the beginning and end, drops the middle) to stay within LLM context windows.

94|## Quick Start
95|
96|### Prerequisites
97|
98|- Docker and Docker Compose
99|- Python 3.10+ (for the MCP bridge)
100|- Hermes Agent, Claude Code, or any MCP-compatible client
101|
102|### 1. Clone
103|
104|```bash
105|git clone https://github.com/vanmanhacks/other-memory.git
106|cd other-memory
107|```
108|
109|### 2. Generate secrets
110|
111|```bash
112|cp .env.example .env && chmod 600 .env
113|openssl rand -hex 32  # → paste into SEARXNG_SECRET in .env
114|openssl rand -hex 16  # → paste into CRAWL4AI_TOKEN in .env
115|```
116|
117|### 3. Create docker secret files
118|
119|```bash
120|grep SEARXNG_SECRET .env | cut -d= -f2 > searxng_secret && chmod 600 searxng_secret
121|grep CRAWL4AI_TOKEN .env | cut -d= -f2 > crawl4ai_token && chmod 600 crawl4ai_token
122|```
123|
124|### 4. Start the stack
125|
126|```bash
127|docker compose up -d
128|```
129|
130|Verify everything is healthy:
131|
132|```bash
133|docker compose ps            # All three services should show "healthy"
134|curl -s 'http://localhost:8888/search?format=json&q=test' | jq '.results | length'
135|curl -s http://localhost:11235/health
136|docker compose exec redis redis-cli -p 6379 PING
137|```
138|
139|### 5. Register with Hermes Agent
140|
141|```bash
142|hermes mcp add other-memory --command /path/to/other-memory/metamcp/run.sh
143|```
144|
145|Restart your Hermes session. The `web_search` and `web_crawl` tools will now appear in your agent's toolset.
146|
147|**For Claude Code:**
148|
149|```bash
150|# Add to your Claude Code MCP config:
151|claude mcp add other-memory --command /path/to/other-memory/metamcp/run.sh
152|```
153|
154|**For any generic MCP client (SSE transport):**
155|
156|```bash
157|python3 -m metamcp.server --sse 8765
158|# Connect your client to http://localhost:8765/mcp
159|```

163|## NixOS Deployment
164|
165|Included is `other-memory.nix`, a NixOS module that manages the entire stack as systemd services.
166|
167|### Systemd Architecture
168|
169|```
170|other-memory-secret.service         (oneshot; generates secrets at boot)
171|         │
172|         ▼
173|other-memory-stack.service          (oneshot; docker compose up/down)
174|```
175|
176|### Integration
177|
178|```nix
179|# In your flake.nix or configuration.nix:
180|imports = [ /path/to/other-memory/other-memory.nix ];
181|```
182|
183|**What it does:**
184|
185|| Service | Purpose |
186||---------|---------|
187|| `other-memory-secret` | Generates `/etc/other-memory/searxng_secret`, `/etc/other-memory/crawl4ai_token`, and `/etc/other-memory/env` on first boot. Idempotent; never overwrites existing secrets. |
188|| `other-memory-stack` | Runs `docker compose up -d` / `docker compose down`. Requires Docker and `other-memory-secret`. Remains after exit (`RemainAfterExit=true`) so the stack stays running. |
189|
190|**Secrets live at:** `/etc/other-memory/` (root-owned directory, 600 files). The env file is automatically populated with `SEARXNG_SECRET` and `CRAWL4AI_TOKEN`.
191|
192|**On non-NixOS systems:** use the `.env` file approach. Same variables, same effect.

196|## MCP Tools Reference
197|
198|### `web_search`
199|
200|Search the web through ten upstream engines.
201|
202|| Parameter | Type | Required | Default | Description |
203||-----------|------|----------|---------|-------------|
204|| `query` | string | ✅ | N/A | Search query. Use technical terms, CVE IDs, package names. |
205|| `num_results` | integer | N/A | 10 | Results to return (1–20). |
206|| `language` | string | N/A | `en` | Language code: `en`, `de`, `fr`, `ja`, `zh`… |
207|| `categories` | array | N/A | `["general"]` | One or more: `general`, `news`, `science`, `files`, `images`, `social_media`. |
208|
209|**Response fields:**
210|
211|```jsonc
212|{
213|  "query": "searxng docker setup",
214|  "total_results_raw": 47,           // Before dedup
215|  "total_results_deduped": 31,       // After URL dedup
216|  "results": [
217|    {
218|      "title": "SearXNG Documentation",
219|      "url": "https://docs.searxng.org/",
220|      "snippet": "SearXNG is a free internet metasearch engine…",
221|      "engines": ["google", "duckduckgo", "bing"]  // Which engines found this
222|    }
223|  ],
224|  "engines_queried": ["bing", "brave", "duckduckgo", …],
225|  "query_time_ms": 847,
226|  "cached": false,                   // true = returned from Redis cache
227|  "timestamp": "2026-06-29T22:12:41.417141+00:00"
228|}
229|```
230|
231|**Note:** SafeSearch is intentionally set to `0` (disabled). This is the correct default for security research: CVEs, exploit PoCs, and offensive security tools are often flagged by content filters. If you need filtered results for general use, modify `safesearch` in `metamcp/server.py` line 244.
232|
233|### `web_crawl`
234|
235|Extract content from any URL with full JavaScript rendering.
236|
237|| Parameter | Type | Required | Default | Description |
238||-----------|------|----------|---------|-------------|
239|| `url` | string | ✅ | N/A | Full URL including `https://`. |
240|| `render_js` | boolean | N/A | `true` | Execute JavaScript before extraction. Required for React/Vue/Angular. Set `false` for static HTML (faster). |
241|| `extract_mode` | string | N/A | `markdown` | Output format: `markdown`, `text`, or `html`. |
242|
243|**Response fields:**
244|
245|```jsonc
246|{
247|  "url": "https://example.com",
248|  "title": "Example Domain",
249|  "content": "# Example Domain\n\nThis domain is for…",  // Markdown/Text/HTML
250|  "content_length": 425,
251|  "original_length": 425,            // Pre-truncation length
252|  "truncated": false,                // true if content exceeded 50K chars
253|  "links_found": 12,                 // Count of discovered URLs
254|  "rendered": true,                  // Whether JS was executed
255|  "extract_mode": "markdown",
256|  "crawl_time_ms": 2341,
257|  "cached": false,
258|  "timestamp": "2026-06-29T22:12:43.518000+00:00"
259|}
260|```
261|
262|**Security:** The bridge blocks internal IPs, metadata endpoints (AWS/GCP/Azure), and non-HTTP schemes. This is not a vulnerability scanner; it refuses to crawl `localhost`, `169.254.169.254`, `file://`, etc.

266|## Configuration Reference
267|
268|### Environment Variables
269|
270|| Variable | Required | Generated By | Description |
271||----------|----------|-------------|-------------|
272|| `SEARXNG_SECRET` | Yes | `openssl rand -hex 32` | 32-byte hex secret for SearXNG sessions/CSRF. |
273|| `CRAWL4AI_TOKEN` | Yes | `openssl rand -hex 16` | Internal API token for Crawl4AI (localhost only). |
274|
275|### Docker Compose Services
276|
277|| Service | Image | Port | Memory | Purpose |
278||---------|-------|------|--------|---------|
279|| SearXNG | `searxng/searxng:2026.6.4` | `127.0.0.1:8888` | ~200 MB | Meta search engine |
280|| Crawl4AI | `unclecode/crawl4ai:0.8.9` | `127.0.0.1:11235` | 2 GB | JS-rendered crawler |
281|| Redis | `redis:7-alpine` | `127.0.0.1:6380` | 256 MB | Query cache (LRU eviction) |
282|
283|All services bind to `127.0.0.1`; nothing is exposed to the network. The MCP bridge communicates over localhost, so your search traffic stays private.
284|
285|### Port Map
286|
287|| Port | Service | Conflict Check |
288||------|---------|----------------|
289|| `8888` | SearXNG | None (if you run GHOLA: GHOLA doesn't use this) |
290|| `11235` | Crawl4AI | None |
291|| `6380` | Redis | Separate from GHOLA Redis (`6379`) |
292|
293|### SearXNG Settings
294|
295|The SearXNG configuration at `searxng/settings.yml` enables ten upstream search engines:
296|
297|**General:** Google, DuckDuckGo, Bing, Brave, Mojeek, Qwant, Startpage
298|**News:** Google News
299|**Code/Tech:** GitHub, Wikipedia
300|
301|Engines are queried in parallel. Response time is the slowest responding engine (800ms–2s). SearXNG uses rotating user agents and connection pooling for upstream reliability.
302|
303|The rate limiter (`searxng/limiter.toml`) allows unlimited requests from localhost and the Docker bridge network. Actual rate limiting is handled upstream by each search engine; SearXNG itself imposes minimal overhead.
304|
305|### Redis Cache
306|
307|- **Key format:** `om:search:<sha256-truncated>` or `om:crawl:<sha256-truncated>`
308|- **TTL:** 1 hour (3600 seconds)
309|- **Eviction:** `allkeys-lru`; old entries evicted when 256 MB is reached
310|- **Persistence:** AOF (append-only file); cache survives restarts
311|- **Graceful degradation:** If Redis is unreachable, the bridge operates without caching (no errors, no crashes)

315|## Security
316|
317|### What's Protected
318|
319|- **No API keys required.** None. You never provision a Firecrawl API key, Tavily key, Serper key, or Brave Search API key. SearXNG queries upstream engines the same way a browser would, using rotating user agents from a list of common desktop browsers.
320|- **All services bind to localhost only** (`127.0.0.1`). Nothing is exposed to your network or the internet.
321|- **SSRF protection** in the MCP bridge blocks internal IP ranges, metadata endpoints, and non-HTTP URL schemes. An agent cannot trick the crawler into probing your internal network.
322|- **Rate limiting** at the SearXNG level: 30 requests/minute to upstream engines (configurable in `limiter.toml`).
323|- **Redis cache** amortizes repeated queries; the same search term within an hour hits cache, not Google.
324|- **Secrets** are generated with `openssl rand -hex` and stored in Docker secrets (`searxng_secret`, `crawl4ai_token`) or `/etc/other-memory/` on NixOS.
325|
326|### What's Not Protected (by Design)
327|
328|- **No authentication on SearXNG or Crawl4AI.** They bind to localhost only. If an attacker has localhost access, you have bigger problems.
329|- **No TLS between services.** Same-host communication over localhost doesn't benefit from TLS.
330|- **No audit logging.** This is a utility bridge, not a SIEM. Hermes Agent logs tool calls; SearXNG has its own access logging.

358|## Production Notes
359|
360|This stack has been running 24/7 on a Hetzner VPS (4 GB RAM, 2 vCPUs) alongside GHOLA (an offensive security framework) and other services. Observed performance:
361|
362|- **SearXNG memory:** ~200 MB steady state
363|- **Crawl4AI memory:** 300–600 MB (with Playwright headless browser; spikes during concurrent crawls)
364|- **Redis memory:** ~50 MB typical (256 MB cap)
365|- **Total stack footprint:** ~1 GB RAM
366|- **Search latency:** 800ms–2s (parallel upstream query, median ~1.2s)
367|- **Crawl latency:** 1s–8s (JS rendering dominates; static pages are fast)
368|
369|The `restart: unless-stopped` policy on all containers handles Docker daemon restarts. Health checks run every 30–60 seconds.

373|## Upgrading
374|
375|Images are pinned to specific versions in `docker-compose.yml` for reproducible deployments. To upgrade:
376|
377|```bash
378|# 1. Check for newer images
379|docker compose pull
380|
381|# 2. Update the image hash in docker-compose.yml
382|docker compose images --format json | jq -r '.[] | "\(.Service): \(.Image)@\(.ID)"'
383|
384|# 3. Re-deploy
385|docker compose up -d
386|
387|# 4. Verify health
388|docker compose ps
389|```
390|
391|**Before upgrading:** Crawl4AI changes its response format between minor versions. The bridge handles known format variations (v0.4–v0.8.x), but review the [Crawl4AI changelog](https://github.com/unclecode/crawl4ai/releases) before bumping major versions. SearXNG upgrades are safe; the JSON API is stable.

395|## Troubleshooting
396|
397|### SearXNG returns empty results
398|
399|```bash
400|docker compose logs searxng | tail -20
401|```
402|
403|Common causes:
404|- Upstream engines rate-limiting your IP (rare; rotating user agents mitigate this)
405|- `SEARXNG_SECRET` not set or mismatched between `.env` and `searxng_secret` file
406|- DNS issues on the host
407|
408|### Crawl4AI times out
409|
410|```bash
411|docker compose logs crawl4ai | tail -20
412|```
413|
414|Crawl4AI has a 60-second timeout. Large pages or slow JavaScript-heavy sites may time out. The bridge returns a clear error: `"Crawl4ai timed out after 60s"`. Increase the timeout in `server.py` line 366 if needed.
415|
416|### Redis connection refused
417|
418|```bash
419|docker compose exec redis redis-cli -p 6379 PING
420|```
421|
422|If Redis is down, the bridge logs a warning and continues without caching. Fix: `docker compose restart redis`.
423|
424|### "MCP server not found" in Hermes
425|
426|```bash
427|hermes mcp list
428|```
429|
430|Verify the path in your `hermes mcp add` command is absolute and the `run.sh` is executable:
431|
432|```bash
433|chmod +x /path/to/other-memory/metamcp/run.sh
434|hermes mcp add other-memory --command /path/to/other-memory/metamcp/run.sh
435|```
